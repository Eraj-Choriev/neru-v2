02

Критичные фичи — нужно сейчас

🔋
"Я приехал на станцию — она занята. Я хочу узнать когда освободится, не обновляя страницу каждую минуту."
— типичный пользователь EV приложения
📱
Критично
PWA — установка на телефон
Пользователи с EV используют приложение стоя у станции. Открывать браузер → набирать URL каждый раз — огромный friction. PWA решает это за одно нажатие.
Почему важно
Retention падает на 40% если нет иконки на экране. Пользователи EV заходят часто и срочно — иконка на рабочем столе критична.
manifest.json service-worker.js Cache API BeforeInstallPrompt
Усилия:
2–3 часа
⭐
Критично
Избранные станции
Большинство пользователей заряжаются у дома, офиса или торгового центра. Им нужен мгновенный доступ к "своим" 2–3 станциям без поиска каждый раз.
Почему важно
Повторный сценарий использования — самый частый. Без избранного каждый визит начинается с поиска. Это раздражает.
localStorage heart icon UI quick access bar
Усилия:
3–4 часа
🔔
Критично
Push-уведомления о статусе
Если любимая станция занята — пользователь хочет получить уведомление когда она освободится. Сейчас нужно вручную обновлять страницу.
Почему важно
Это разница между "приложение работает для меня" и "я работаю для приложения". Автоматизирует самый болезненный момент ожидания.
Notification API Service Worker polling в фоне
Усилия:
4–6 часов
💰
Критично
Калькулятор стоимости зарядки
Пользователь видит тариф, но не понимает сколько это будет стоить для его конкретного авто с текущим % батареи до нужного уровня.
Почему важно
Цена — ключевой фактор выбора станции. Сейчас данные есть, но их нужно считать в уме. Добавить поле "мой авто / % батареи → стоимость".

Высокий приоритет — следующий спринт

📋
Высокий
История поездок
Журнал: когда, куда, сколько заряжал, сколько потратил. Пользователь может отслеживать расходы на зарядку за месяц.
Почему важно
Владельцы EV считают затраты на зарядку vs бензин. Это данные, которые они хотят видеть. Добавляет ценность каждого сеанса зарядки.
localStorage history chart.js / SVG monthly stats
Усилия:
5–8 часов
👋
Высокий
Onboarding-тур для новых пользователей
Первый запуск: краткое знакомство с интерфейсом. Подсвечивает кнопки, объясняет значения цветов станций, показывает как найти ближайшую.
Почему важно
Цветовое кодирование станций не очевидно с первого взгляда. Новый пользователь должен понять приложение за 30 секунд, иначе уйдёт.
spotlight overlay step tooltips localStorage seen
Усилия:
3–4 часа
🔗
Высокий
Поделиться станцией
Скопировать ссылку на станцию или отправить другу. URL типа neru.app/?station=42 открывает карту с нужной станцией.
Почему важно
Органический рост — пользователи делятся станциями в Telegram/WhatsApp. Каждый шер — это бесплатный маркетинг и новый пользователь.
URL params navigator.share() clipboard API
Усилия:
1–2 часа
📍
Высокий
Поиск по адресу / радиусу
Ввести адрес и найти станции рядом с ним — например "найти у работы", не у текущего местоположения. Планирование маршрута заранее.
Почему важно
Пользователь часто планирует зарядку заранее — перед поездкой на работу. Геолокация сейчас ограничивает "где я стою сейчас".

Расчётное время ожидания
Если станция занята — показать приблизительное время до освобождения на основе среднего времени зарядки и текущего charge_level коннектора.

charging_level уже есть в API. Если коннектор на 80% — значит скоро завершит. Простая эвристика снимает главную тревогу пользователя у занятой станции.

PWA — технические детали

PWA — наиболее высокоприоритетное улучшение с минимальными усилиями. Три файла полностью трансформируют пользовательский опыт.

01
manifest.json
Название, иконки, цвета, display: standalone. Позволяет установить на экран.
02
service-worker.js
Cache-first для CSS/JS, network-first для API. Работает офлайн с кешем.
03
Install prompt UI
Баннер "Установить приложение" при первом визите — конверсия в установку.
